# 🏗 Architecture

This document provides a simple, beginner‑friendly overview of the architecture used in this project.  
The goal is to clearly explain how the system works, which AWS services are involved, and why this design is ideal for learning cloud fundamentals.

---

## 🧭 Architecture Overview

This project uses one of the simplest AWS hosting patterns:

**User → S3 Static Website Hosting**

The website files (HTML, CSS, JS) are stored in an Amazon S3 bucket configured for static website hosting.  
Users access the site directly through the S3 website endpoint.

![Architecture Diagram](../diagrams//high-level-architecture.png)

---

## 🏛 Architecture

At a high level, the architecture consists of:

- **Amazon S3** hosting the static website  
- **Public website endpoint** generated by S3  
- **Bucket Policy** controlling access  
- **No servers, no backend, no databases** — perfect for beginners  

This architecture is intentionally simple to help build foundational AWS knowledge.

---

## ☁️ AWS Services Used

- **Amazon S3** — stores and serves static website files   

For this beginner version, only S3 is required.

---

## 🔁 End‑to‑End Flow

Here’s how a user request moves through the system:

1. User enters the S3 website endpoint URL in their browser  
2. Browser sends an HTTP request to the S3 website endpoint  
3. S3 retrieves the requested file (e.g., `index.html`)  
4. S3 returns the file to the user’s browser  
5. Browser renders the website  

This simple flow makes it easy to understand how static hosting works.

---

## ⭐ Key Architecture Benefits

- **Beginner‑friendly** — easy to set up and understand  
- **Low cost** — S3 static hosting is extremely inexpensive  
- **No servers** — nothing to patch, update, or maintain  
- **Fast deployment** — upload files and the site is live  
- **Scalable** — S3 automatically handles traffic  

This architecture is widely used for portfolios, landing pages, and learning projects.
